# Nala brand usage and product-led social system audit

## Live-product finding

Nala already expresses most of the approved brand system through native application code rather than imported image files.

### Visible now

- Lowercase `nala.` wordmark in the navigation and footer.
- Approved Ink, Forest, Deep, Leaf, Lime, Sand, Paper, Sun and Clay palette.
- Serif-led editorial headlines and sans-serif interface copy.
- Product-derived visual language: first-work journey board, progress track, paid task ticket, readiness steps, proof-of-work passport, worker/business bridge, business outcome stack, comparison panel and trust safeguards.
- Current first-opportunity, paid-work, verification and progress messaging.

### Present in the brand package but not used as runtime files

- Exported logo lockups and symbols.
- Platform profile images and banners.
- Social post templates.
- Graphic pathway patterns.
- Campaign artwork.

This separation is intentional. Native HTML and CSS are more accessible, responsive and maintainable than placing flattened social graphics inside product workflows.

### Missing before this release

- Branded browser icon.
- Product-led Open Graph sharing artwork.
- Explicit Open Graph and Twitter card metadata.

## Safe integration completed

This release adds only:

- `src/app/icon.svg`
- `public/nala-social-share.svg`
- browser, Open Graph and Twitter metadata in `src/app/layout.tsx`

No landing-page composition, workspace layout, route, workflow, simulation, permission, database schema or business rule is changed.
